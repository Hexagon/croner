(constant) maxDelay :number

Many JS engines stores the delay as a 32-bit signed integer internally. This causes an integer overflow when using delays larger than 2147483647, resulting in the timeout being executed immediately.

All JS engines implements an immediate execution of delays larger that a 32-bit int to keep the behaviour concistent.
